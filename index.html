Systems Thinking

a way of understanding a problem that emphasizes the relationships among a system's parts, rather than the parts themselves. As an approach, it enables long-view solutions, seeking to identify unintended consequences.

Horizontal Software Development

is an approach to software development that emphasizes layers of functionality. Layers can be built upon, reconfigured, and independently evolved to address current and emerging needs.

Horizontal Software Development

is not a recipe, set of hard-and-fast rules, discipline, etc. It is rather a set of principles, an approach, that can be applied in many instances and in many guises in order to allow certain (desirable) properties to emerge.

Desirable Properties

  • Separation of Concerns
  • Clarity/Simplicity
  • Flexibility/Evolvability
  • Dependency Management

Why is horizontal development particularly difficult in libraries? Because libraries are (historically) based on a different set of assumptions & guiding principles: verticality, expertise, top-down order & structure, resting on unchanging technologies and realities.
